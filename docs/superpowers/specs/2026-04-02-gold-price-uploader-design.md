# 金価格自動アップロードツール 設計書

## 概要

毎日の金価格を田中貴金属サイトから取得し、お客様が確認・修正した上で、WordPressサイトとGoogleビジネスプロフィールに投稿するWebツール。

## ユーザーフロー

1. お客様がブラウザでツールにアクセス（Basic認証）
2. 「価格を取得する」ボタンを押す
3. 田中貴金属サイトから金価格がスクレイピングされ、画面に表示される
4. 必要に応じて価格を手直しする
5. 「アップロード」ボタンを押す
6. WordPressの固定ページ更新 + Googleビジネスプロフィール投稿が実行される
7. 結果が画面に表示される

## 投稿ルール

### WordPress
- 指定した固定ページ1つを毎回上書き更新する
- 常に最新の金価格だけが表示される

### Googleビジネスプロフィール
- 毎日新規投稿を作成する
- 同じ日に2回目以降の実行時は、当日の既存投稿を削除して再投稿する

## 技術構成

### サーバー構成

```
VPS（Ubuntu）
├── Nginx（リバースプロキシ + Basic認証 + HTTPS）
└── Flask アプリ（Gunicornで起動）
    ├── /[ランダムURL]/        → 操作画面
    ├── /[ランダムURL]/fetch   → 金価格スクレイピングAPI
    └── /[ランダムURL]/upload  → WP・Google投稿API
```

### 使用ライブラリ

| 用途 | ライブラリ |
|------|-----------|
| Webフレームワーク | Flask |
| スクレイピング | requests + BeautifulSoup |
| WordPress API | requests（REST API直接呼び出し） |
| Google投稿 | Google Business Profile API（google-auth等） |
| プロセス管理 | Gunicorn + systemd |

### データソース

- 金価格取得元: https://gold.tanaka.co.jp/commodity/souba/d-gold.php

## セキュリティ

### アクセス制御
- HTTPS必須（Let's Encryptで無料SSL証明書、自動更新）
- Basic認証（Nginxレベルで設定）
- 管理画面のURL難読化（ランダムなパスを使用）

### API認証情報の保護
- WordPress APIキー、Google API認証情報等は環境変数で管理
- `.env`ファイルで管理し、gitにはコミットしない

### アプリケーション保護
- セキュリティヘッダー（Nginxで設定）
  - X-Content-Type-Options
  - X-Frame-Options
  - Content-Security-Policy
- 入力バリデーション（価格フィールドは数値のみ）

### サーバー保護
- SSH鍵認証のみ（パスワードログイン無効化）
- ファイアウォール（ufw）— 80/443/SSHのみ開放
- 自動セキュリティアップデート（unattended-upgrades）

## デプロイ

- systemdでFlaskアプリを自動起動（サーバー再起動時も自動復帰）
- NginxでHTTPS終端 + Basic認証
- Let's EncryptでSSL証明書（certbot自動更新）
